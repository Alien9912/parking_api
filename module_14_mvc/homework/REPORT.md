# Отчёт по повышению зрелости API (задача 2)

## Текущий уровень API (из задачи 1)
- Используются разные URL: `/get-room`, `/add-room`, `/booking`.
- Применяются GET и POST, статус-коды 200, 400, 409.
- Уровень по модели Ричардсона: **1** (с элементами уровня 2).

## Недостатки
- Имена эндпоинтов содержат глаголы (`get-room`, `add-room`), не соответствуют REST.
- Для бронирования используется POST, хотя операция идемпотентна – лучше DELETE.
- Нет гиперссылок (HATEOAS), клиент вынужден сам собирать URL.

## Предлагаемые улучшения (уровень 2–3)
| Было | Стало |
|------|-------|
| GET /get-room | GET /rooms |
| POST /add-room | POST /rooms |
| POST /booking с телом {roomId} | DELETE /rooms/{roomId} |
| 200 OK при бронировании | 204 No Content |
| Нет ссылок | Добавить поле `links` в ответы |

## Пример HATEOAS-ответа для GET /rooms
```json
{
  "rooms": [
    {
      "roomId": 1,
